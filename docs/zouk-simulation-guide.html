Zouk Simulation Sandbox

用仿真环境快速验证 Zouk 的端到端行为

这页教你用 server/test-support/zouk-simulation.mjs 启动隔离 server，模拟 human、web socket、daemon 和 agent 内部工具调用。

先记模型 直接运行 API 速查 测试套路

先记住一个模型

仿真环境不是 mock 业务逻辑。它会启动真实 server/index.js，只把外部世界换成测试可控的临时目录、随机端口、websocket 客户端和 daemon websocket。

createZoukSimulation 随机端口启动 server，写入 root session，使用临时 config/uploads。
Human createGuest 与 sendHumanMessage 走真实 REST API。
Web Client connectWebClient 接 /ws，等待 init、message、agent 状态。
Daemon connectDaemon 接 /daemon/connect，发送 ready/status/activity。
Agent API agentReceive、agentSend、history、search 验证 agent 视角。

一句话：用 sim 操作真实 Zouk，用 web 观察浏览器会收到什么，用 daemon 假装本机 daemon 和 agent 正在跑。

最快上手

先跑现成契约测试，确认你的环境能启动 sandbox。

Run the simulation contract suite 
node --test server/test-simulation-e2e.mjs
Run all server tests, including simulation 
npm run test:server
看到 3 个 simulation 测试通过就够你开始改代码。

这 3 个测试分别覆盖 human/web 消息、agent internal API、daemon 生命周期与 push delivery。

最小可跑脚本

这个脚本会启动隔离 Zouk、创建用户、创建 machine key、连接 daemon、启动 agent、发消息，并断言 daemon 收到 delivery。

Paste into terminal from repo root 
node --input-type=module <<'EOF'
import assert from 'node:assert/strict';
import { createZoukSimulation } from './server/test-support/zouk-simulation.mjs';

const sim = await createZoukSimulation({ name: 'manual-sim' });

try {
  const alice = await sim.createGuest('alice');
  const web = await sim.connectWebClient({ token: alice.token });
  const init = await web.waitForType('init');
  assert.equal(init.workspaceId, 'default');

  const key = await sim.createMachineKey('manual-daemon');
  const daemon = await sim.connectDaemon({ key: key.rawKey });
  daemon.ready({ runtimes: ['claude'], capabilities: ['workspace_fs'] });
  await sim.waitForMachineReady(key.key.id);

  const agentId = 'agent-manual-smoke';
  const startFrame = daemon.waitForStart(agentId);
  await sim.startAgent({
    agentId,
    name: 'smokebot',
    displayName: 'Smoke Bot',
    machineId: key.key.id,
  });
  await startFrame;
  daemon.agentStatus(agentId, { status: 'active' });

  const delivery = daemon.waitForDelivery(
    agentId,
    (event) => event.message?.content === 'ping @smokebot'
  );
  await sim.sendHumanMessage({
    token: alice.token,
    target: '#all',
    content: 'ping @smokebot',
  });

  const event = await delivery;
  assert.equal(event.message.sender_name, alice.user.name);
  console.log('delivery ok:', event.message.content);
} finally {
  await sim.stop();
}
EOF
  • 1createZoukSimulation 会自动清理临时目录；测试里仍建议用 t.after(() => sim.stop())。
  • 2daemon.ready() 必须在 start agent 前发送，否则 server 找不到支持 runtime 的 daemon。
  • 3要测试 agent 被动收消息，用 daemon.waitForDelivery；要测试 agent 主动拉消息，用 sim.agentReceive。

API 速查

常用动作按对象分组。优先用这些高层 API，只有需要覆盖新服务行为时再补 helper。

对象 方法 用途
sim createGuest(name) 创建 guest session，拿到 token 和 user。
sim sendHumanMessage({ token, target, content }) 通过 POST /api/messages 写真实消息，触发 fanout 和 web broadcast。
sim getMessages({ channel, limit }) 读取 web 视角的 channel history。
web waitForType('init'), waitFor(predicate) 观察浏览器 websocket 会收到的 init、message、agent 状态。
sim createMachineKey(name) 创建真实 machine key，daemon 用返回的 rawKey 连接。
daemon ready(...), agentStatus(...), agentActivity(...) 模拟 daemon 上线、agent 生命周期、agent 活动事件。
daemon waitForStart(agentId), waitForDelivery(agentId) 断言 server 真的向 daemon 下发 start 或 message delivery。
sim agentReceive, agentSend, agentHistory, agentSearch 覆盖 agent internal API，也就是 chat tools 背后的服务契约。

写新测试的套路

消息收发

用 human 写消息，然后同时断言 web broadcast、channel history、agent receive 或 daemon delivery。

sendHumanMessage web.waitFor agentReceive

agent 行为

先创建 machine key 和 daemon，再 start agent。用 status/activity 事件驱动 server 状态变化。

createMachineKey daemon.ready agentStatus

agent 工具 API

不需要真实 agent 进程。直接用 internal API 验证 receive、send、history、search 的服务契约。

agentSend agentHistory agentSearch
Test template 
import { test } from 'node:test';
import assert from 'node:assert/strict';
import { createZoukSimulation } from './test-support/zouk-simulation.mjs';

test('my behavior contract', async (t) => {
  const sim = await createZoukSimulation({ name: 'my-contract' });
  t.after(() => sim.stop());

  const alice = await sim.createGuest('alice');
  const web = await sim.connectWebClient({ token: alice.token });
  await web.waitForType('init');

  const marker = `probe-${Date.now()}`;
  const broadcast = web.waitFor((event) => (
    event.type === 'message' && event.message?.content === marker
  ));

  await sim.sendHumanMessage({ token: alice.token, target: '#all', content: marker });
  assert.equal((await broadcast).message.content, marker);
});

调试规则：如果你在修 server 行为，先只跑 node --test server/test-simulation-e2e.mjs 或你的新文件；通过后再跑 npm run test:server。
