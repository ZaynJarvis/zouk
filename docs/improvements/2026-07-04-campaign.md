## Theme 1 ‚Äî Simulation is now full-stack (the e2e loop)

The sim previously faked the daemon protocol by hand (`SimulatedDaemon`). Now the loop closes: **real server + real zouk-daemon + mock agent runtime**, no LLM CLI needed.

- **zouk-daemon#8 ‚Äî mock runtime driver.** Driver id `mock` (advertised only when `ZOUK_MOCK_RUNTIME=1`). The runtime is a real long-lived process: spawns the actual agent-mcp server, speaks real MCP JSON-RPC ‚Äî `check_messages` (advances the server seen-cursor exactly like a real agent) ‚Üí behavior rules from `<workDir>/mock-behavior.json` (`{rules:[{match, reply, delayMs}], fallbackReply, contextUsagePerTurnPct}`) ‚Üí `send_message`, with one considered retry on send-freshness holds. Emits session_init/text/tool_call/turn_end/context_usage.
- **zouk#410 ‚Äî real-daemon e2e harness + eval runner.** `server/test-support/zouk-real-daemon.mjs` (`startRealDaemon`, `isRealDaemonAvailable`, `writeMockBehavior`; binary via `ZOUK_DAEMON_BIN` or sibling checkout; CI-safe skip). `server/test-daemon-e2e.mjs`: 4 full-stack tests (default ack behavior, custom rules, web events, seen-cursor freshness) ‚Äî stable 3√ó consecutive. `npm run eval` (`server/eval-run.mjs`): reply-storm eval + daemon smoke, scorecard, nonzero exit on failure. **This is the guardrail for all future feature work.**
- Why it matters: with `analyse ‚Üí backlog ‚Üí execute ‚Üí analyse` loops, an agent can now spin the entire product locally, script deterministic agent behavior, and assert outcomes ‚Äî the eval substrate the owner asked for.

## Theme 2 ‚Äî Agent clone (zouk#411 + zouk-daemon#10)

"Clone an agent quickly, let it do its job in a separate clean session, reusing identity/workspace/OV memory, multiplexed alongside the original."

- Design: every runtime keyspace on both sides is strictly 1:1 with agentId (server `daemonSockets`/cursors/tokens; the daemon kills the previous process on a same-id `agent:start`). So a clone is a **derived agent instance**: new internal id/name `<parent>-cN`, but shares the parent's config, machine, **workDir**, and **OpenViking identity** (same `openvikingUserId`+`apiKey` ‚áí same memory namespace; distinct `openvikingSessionId` suffix keeps its pending buffer isolated). Fresh sessionId (clean context), fresh cursors/token, `lifecycle: ephemeral`, `cloneOf` marker.
- Anti-double-reply: clones copy **no** channel subscriptions ‚Äî reachable via DM (and an optional channel passed at clone time). Channel fan-out never hits them by default.
- Lifecycle: explicit stop **dissolves** the clone (config/token/memberships removed; its messages remain). Daemon disconnect does not dissolve. Cap 4 live clones/parent; clone-of-clone rejected; excluded from autoStart.
- Surfaces: `POST /api/agents/:id/clone {prompt?, channel?}` (human), MCP tool `clone` (agents self-clone ‚Üí coordinator fan-out), web Clone button + clone badge. Initial `prompt` arrives as a DM, waking the clone through the normal delivery path.
- Eval: `server/test-agent-clone.mjs` ‚Äî 10 tests (identity inheritance, DM isolation both directions, channel non-delivery, dissolve, cap, name allocation, prompt delivery).
- Known follow-up: @mentioning a clone in a channel does NOT deliver to it (router requires membership) ‚Äî DM it, or pass `channel` at clone time. See "Deferred".

## Theme 3 ‚Äî Coordination (zouk#409 + zouk-daemon#9)

From OSS research (A2A agent-cards, OpenAI handoffs, CrewAI delegation, blackboard papers), the two best chat-native fits:

- **Agent cards**: the `directory` tool now returns per-agent description, runtime/model, live status/activity, **claimedTasks**, and subscribed channels ‚Äî enough for an agent to pick a collaborator by capability and current workload.
- **Delegate with result contract**: `task create` accepts `assignee`. The task is pre-claimed for the assignee (reuses claim mechanics ‚Äî no new states), the assignee is woken by a DM referencing the task, and when they move it to `in_review`/`done` the creator gets an automatic completion DM. Delegation is now one tool call with a built-in report-back.
- Eval: `server/test-delegation.mjs` (assign ‚Üí wake ‚Üí complete ‚Üí notify loop, directory enrichment, backward compat).

## Theme 4 ‚Äî Verified bug fixes

Every fix was first confirmed by an independent skeptic agent with code evidence; refuted claims were dropped (F8 "activity duplication on reconnect" ‚Äî mechanisms don't exist; F19 "cached-stop sends inactive" ‚Äî correct for explicit stops; original F2 "OV context race" ‚Äî server sends context before delivery on the same socket, ordering is guaranteed).

zouk:
- **#402** ‚Äî de-flaked `test-workspace-bounce.mjs` (the "alice must receive an init payload" flake). Root causes: hardcoded ports colliding under parallel `node --test`, missing `ZOUK_NO_MOCK`, and a WS listener attach race. Ported to the sim harness; 10√ó stable.
- **#403** ‚Äî `ov-mcp-proxy`: MCP sessions were keyed by (url,user) so two agents sharing an OV user shared one session (interleaving corruption ‚Äî also a clone blocker); tool cache was global across OV URLs; sessions never evicted. Now keyed per credential (hashed), cached per URL, TTL-evicted.
- **#404** ‚Äî attachment prune was age-only: a blob referenced by a 13-day-old message died at day 14. Prune now protects ids referenced by messages (in-memory ‚à™ DB refs).
- **#405** ‚Äî `POST /internal/agent/:id/send` had no rate limit (fresh UUID per send defeats dedupe). Sliding window 20/10s per agent (env-tunable), 429 + Retry-After. Reply-storm eval unaffected.
- **#406** ‚Äî web: `WsInitEvent` was missing 6 fields the store reads; missing event types; `workspace:file_tree` was silently overwriting saved config state (and the server was persisting a scan's workDir into agent config). Types fixed at the source; config pollution removed on both sides.
- **#407** ‚Äî ready-race: a second daemon's `ready`/`agent:status` could steal a live agent binding, orphaning the first daemon's process. Live-owner guard (skip adoption when the current owner socket is OPEN); autoStart also skips `starting` agents. Daemon-swap behavior preserved (test-daemon-swap green).
- **#408** ‚Äî persistence + delivery bookkeeping: rare-path DB writes (channels, channel/workspace memberships) are now awaited with record-identity error context (hot message path stays fire-and-forget **by design**); `deliverToAllAgents` fan-out counts were always 0 (async result checked synchronously) ‚Äî now `Promise.all` with real counts; pending-delivery replay re-checks subscription so an agent unsubscribed-while-offline no longer gets stale deliveries.

zouk-daemon:
- **#5** ‚Äî CI: there was **no test gate at all**; publish didn't run tests. Added `ci.yml` (PRs + main) and `npm test` in publish.
- **#6** ‚Äî lifecycle races: delivery-drop window while an agent is mid-spawn (buffered now via `startingAgents`); `pendingStops` identity-guard; **cached-idle agents were unreachable after daemon reconnect** (`ready.runningAgents` omitted idle-cache ids ‚Äî the server kept them "active" with no socket binding; found while refuting F19); workspace RPC now uses the agent's configured workDir instead of always the default dir.
- **#7** ‚Äî agent-mcp HTTP client now retries 429 honoring Retry-After (pairs with #405); droid `--model` was silently ignored (restored with validation + warning); codex late token-usage misattribution heuristic.
- **#9/#10** ‚Äî assignee param forwarding; clone tool route.
